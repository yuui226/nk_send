# Z传 — 遥控 / 预览 / 大文件 真机验证清单

> 覆盖分支 `feature/remote-control-lab` 的新功能：无线遥控页、FHD 预览与手势缩放、大文件分块传输与断点续传。
> 每项含**操作**、**预期**。文末「设计如此，非 Bug」列出容易误报的预期表现。
>
> 图例：⬜ 待测　✅ 通过　❌ 不符（记录现象 + 开发者面板日志）

---

## 1. 进入遥控页 / 让路规则

### 1.1 入口按钮
- 操作：文件列表滚到**最顶部** → 看左下角圆钮（取景框图标）。
- 预期：仅在列表顶部时出现；往下翻一点即缩小消失（防翻页误触）。
- ⬜

### 1.2 传输中禁止进入
- 操作：有任务正在传输/等待时，点左下角遥控入口。
- 预期：图标压暗，点击弹提示「传输进行中，完成后再进入监看」，**不进入**。
- ⬜

### 1.3 进入速度
- 操作：空闲（无传输）时点遥控入口。
- 预期：几百毫秒内出现第一帧监看画面；读数条（曝光补偿/ISO/光圈/快门）快速点亮。
- ⬜

### 1.4 首次进入提示
- 操作：首次进入遥控页。
- 预期：顶部一次性提示「遥控期间相机机身暂不可操作」，约 3s 后消失；以后不再提示。
- ⬜

## 2. 监看画面

### 2.1 实时监看
- 操作：进入遥控页，观察画面。
- 预期：实时取景，右下角显示帧率（默认开）；相机不会因监看而很快息屏（若息屏见「非 Bug」）。
- ⬜

### 2.2 HD / FPS 开关
- 操作：点顶栏 **HD**、**FPS** 小按钮切换。
- 预期：HD 开 → 画面切 XGA（更清晰，帧率略降）；FPS 开/关 → 右下角帧率显隐；激活态蓝色高亮。
- ⬜

### 2.3 信号按钮（左上）
- 操作：看左上角信号按钮。
- 预期：在线显示信号格、点击展开 dBm；断开显示红色断连图标（与文件列表同一个）。
- ⬜

### 2.4 曝光模式徽标
- 操作：看取景器左上角。
- 预期：显示当前模式（M/A/S/P），只读（物理拨盘机身无法远程切换）。
- ⬜

### 2.5 RGB 直方图
- 操作：点顶栏直方图图案按钮，再次点击关闭；分别在普通和 HD 监看下观察。
- 预期：取景器左下显示 RGB 叠加直方图，画面变化时同步更新；按钮激活态蓝色高亮；关闭后直方图消失且不再进行逐帧统计。
- ⬜

### 2.6 构图参考线
- 操作：连续点击顶栏网格图案按钮。
- 预期：按“关闭 → 3×3 九宫格 → 4×4 十六宫格 → 关闭”循环；参考线只覆盖实际画面，不伸进 ContentScale.Fit 产生的黑边；非关闭状态按钮蓝色高亮。
- ⬜

### 2.7 对焦方式徽标
- 操作：在相机端切换 AF-S / AF-C / AF-A / AF-F / MF（以机身当前拍照或录像模式可用项为准）。
- 预期：取景器左上与曝光模式并排显示，例如 `M · AF-S`；属性不可读的机型只保留曝光模式，不显示猜测值；开发者日志记录属性码与原始值。
- ⬜

### 2.8 横屏监看
- 操作：点顶栏旋转图案进入横屏，再点横屏工具轨中的同一按钮回竖屏；横屏中调参数、半按对焦、拍照/录像后退出监看。
- 预期：旋转图案平滑转向；工具按钮由横向顶栏变为竖向工具轨，取景器、2×2 参数与快门均完整可操作；退出监看后恢复进入前的屏幕方向策略。
- ⬜

### 2.9 点按画面对焦
- 操作：在 AF-S 下先点距离过近、无法合焦的物体，再点可正常合焦的物体；竖屏、横屏和 HD 各试一次。
- 预期：对焦框准确出现在点击位置；进行中为蓝色，未合焦变红，合焦成功变绿并轻震；点击画面外黑边不触发。
- 日志：首行包含 `tap AF point=(x,y) frame=WxH`；失败包含 `tap AF out of focus`，成功包含 `tap AF locked`，且两者都有 `polls` 与 `elapsed`。
- 操作：将相机切到 MF 后点画面。
- 预期：不发送 AF 命令，顶部提示需切换到自动对焦。
- ⬜

## 3. 参数调节（2×2 直控胶囊）

### 3.1 顺序与布局
- 预期：第一排 曝光补偿 / ISO，第二排 光圈 / 快门速度。每个胶囊 `[− 值 +]`。
- ⬜

### 3.2 单击 / 长按连调
- 操作：点某胶囊 `−`/`+` 一次；再长按不放。
- 预期：单击调一档；长按约 0.4s 后连续调档；画面曝光随之变化（有震动设置时每档轻震）。
- ⬜

### 3.3 弹全表直跳
- 操作：点胶囊**中间数值**。
- 预期：弹出完整档位表（缩放淡入），当前值蓝色高亮、自动滚到附近；点选即设、弹窗淡出消失。
- ⬜

### 3.4 只读参数
- 操作：机身拨到 A 档，看快门胶囊；拨到 S 档看光圈胶囊。
- 预期：不可调的参数整条压暗 + 锁图标，`−/+` 无效。
- ⬜

### 3.5 机身侧改动同步
- 操作：在相机机身上转拨盘改参数。
- 预期：手机读数条跟着变（DevicePropChanged 事件驱动刷新）。
- ⬜

## 4. 对焦 + 拍摄（两段式快门）

### 4.1 半按对焦
- 操作：**按住**快门键不放（镜头/机身在 AF 挡）。
- 预期：中央出现对焦框（蓝色）；只启动一次 AF 并等待最终结果；**合焦成功框转绿 + 轻震一下**（有震动设置时）。
- ⬜

### 4.2 松开拍摄 / 滑出取消
- 操作：按住对焦后，① 在键内松手；② 滑出按钮再松手。
- 预期：① 若 AF 仍在进行，先等待 DeviceReady 终态再拍摄（快门触发轻震，键转圈=等待对焦/拍摄确认）；② **不拍**，但已启动的 AF 协议会收完终态，不留下错位响应。
- ⬜

### 4.3 拍摄确认
- 操作：拍一张，看快门键。
- 预期：触发后转圈（正在等相机确认拍好），收到 ObjectAdded 即停；**不显示缩略图回显**（拍的照片去照片列表看）。
- ⬜

### 4.4 对焦失败
- 操作：对着无法合焦的场景按住快门。
- 预期：对焦框保持蓝色不转绿、不震；开发者日志包含 `AF out of focus`。松手不拍（若滑出）或拍一张（键内松手，允许脱焦拍）。
- ⬜

## 4A. 录像模式（快门键随实体拨杆变身）

> 实现基于 0xD1A6 LiveViewSelector（拨杆位置）与 0x920A/0x920B（开始/结束录像）。
> 2026-07-14 首轮实测：模式检测 ✅、相机侧开录的状态跟踪 ✅、App 停止 ✅；
> App 开录被拒 → 已加放行序列（0xD1F0 属性 → 0x9435 操作码，带恢复）待复测；
> 录像位参数读不到 → 已换录像侧独立参数组（0xD1A8~0xD1AB）待复测。

### 4A.1 模式检测 + 参数组切换
- 操作：遥控页开着，把机身照片/录像拨杆拨到录像位，再拨回来。
- 预期：快门键内芯在 3s 内变红（录像位）/变回白色（照片位）；拨杆在进页前就位于录像位时进页即红。**切到录像位后 2×2 参数换成录像侧一套（可与照片侧数值不同），拖动/点全表可调；拨回照片位恢复照片侧参数。**
- ⬜

### 4A.2 开始/停止录像
- 操作：录像位下，键内松手一次开始，再一次停止。
- 预期：开始后内芯红圆缩成圆角方块（停止键）、取景器右上角出现脉动红点+时长；停止后恢复红圆、徽标消失；相机存卡里有对应视频，时长与徽标计时基本一致。首次开录日志可能出现 `ApplicationMode=1 resp=0x2001`（放行序列，正常）。**停止后检查机身不滞留在应用模式**（拨杆离开录像位/停录后机身按键恢复正常锁定状态，无额外异常）。
- ⬜

### 4A.3 录像中半按对焦
- 操作：录制中按住快门键。
- 预期：与拍照一致——对焦框出现、合焦转绿+轻震；录制不中断。
- ⬜

### 4A.4 滑出取消
- 操作：录像位下按住后滑出按钮再松手（录制中与非录制中各试一次）。
- 预期：既不开始也不停止录像，只结束对焦。
- ⬜

### 4A.5 无法开录的反馈
- 操作：取出存储卡（或制造禁止条件），键内松手尝试开录。
- 预期：顶部提示「无法开始录像，请检查存储卡与相机状态」；开发者日志有 `MovRecProhibit=0x…` 或 `StartMovieRec resp=0x…`。
- ⬜

### 4A.6 相机自行停录
- 操作：录制中让相机自己停（拨回照片位/等卡满/机身限制）。
- 预期：App 侧 3s 内（事件 0xC108/0xC105 或拨杆兜底轮询）自动恢复成非录制状态，计时徽标消失。
- ⬜

## 6. FHD 预览 + 手势缩放

### 6.1 加载与动画
- 操作：文件列表长按一张照片进入全屏预览。
- 预期：先出缩略图（略糊）→ 顶部极细进度条表示加载高清 → 高清淡入变清晰（blur-up）→ 到位**轻震一下**（有震动设置时）。
- ⬜

### 6.2 翻页预取
- 操作：左右翻页浏览。
- 预期：邻居页已预取、翻过去基本不等；连续翻几十张内存稳定不涨（±2 窗口 + RGB_565）。
- ⬜

### 6.3 光圈显示
- 操作：预览一张有 EXIF 的照片，看底栏参数。
- 预期：光圈/快门/ISO/焦距都显示（光圈优先 TAG_F_NUMBER）。若仍缺光圈，记录并发日志。
- ⬜

### 6.4 捏合缩放
- 操作：双指捏合放大/缩小。
- 预期：以捏合中心为不动点平滑缩放，上限约 4x；缩到 1x 以下回弹到 1x。
- ⬜

### 6.5 平移
- 操作：放大后单指拖动。
- 预期：平移画面，到图片边缘停住（不拖出黑边）。
- ⬜

### 6.6 双击缩放
- 操作：在某点双击；再双击。
- 预期：1x → 以该点为中心放大到 2.5x（带动画）；再双击 → 复原到 1x。
- ⬜

### 6.7 缩放与翻页/关闭的关系
- 操作：放大状态下横向拖；缩回 1x 后横滑；1x 单击。
- 预期：**放大时横向拖是平移、不翻页**；缩回 1x 后横滑才翻页；1x 单击关闭预览（放大时单击不关）。
- ⬜

### 6.8 翻页复位
- 操作：放大某张后缩回 1x、翻到下一张、再翻回来。
- 预期：每张进入时都是原始 1x（离页自动复位）。
- ⬜

### 6.9 FHD 不支持机型（若有）
- 操作：换一台可能不支持 0x920F 的机型长按预览。
- 预期：第一页失败后，之后每页**直接显示缩略图、不再页页卡几秒**（会话级熔断）。
- ⬜

## 7. 大文件分块传输 / 断点续传（重点，易出静默损坏）

### 7.1 大文件正常传输
- 操作：传一个 >128MB 的视频。
- 预期：分块下载（GetPartialObjectEx），完成后文件可正常播放、大小正确。
- ⬜

### 7.2 中断续传（核心）
- 操作：传 >128MB 视频到一半 → 断网/停止 → 重连 → 重试。
- 预期：从块边界续传（不从头）；**最终文件可正常播放、大小与原文件一致**（不是错位/损坏）。
- ⬜

### 7.3 >4GB 文件
- 操作：传一个 >4GB 视频；中途中断再续。
- 预期：不被误判「已完整」提前改名；续传后完整可播。
- ⬜

### 7.4 同名不同文件
- 操作：两个不同文件夹里同名（如 DSC_0001.MOV）、内容不同的文件都入队，一个失败后重试。
- 预期：不会把两份数据拼接成损坏文件（半成品按内容身份匹配）。
- ⬜

### 7.5 掉线不假死
- 操作：大文件传输中直接断电/走出范围（无 TCP RST）。
- 预期：约 60s 内该任务失败、可重试；**App 不卡死**（不会抱着锁冻住整个相机管线）。
- ⬜

## 设计如此，非 Bug

- **遥控期间机身被锁**：相机屏显示「已连接」、按键/取景器不可操作——SnapBridge 遥控同样如此。
- **拍照不在遥控页回显**：拍的照片只进相机存卡，要看得返回照片列表（那边会读到新照片）。有意为之，避免占用取图通道。
- **相机自动息屏**：Z 系列待机计时器不被 PTP 流量重置。若长时间监看后相机睡，去相机菜单把「电源关闭延迟 → 待机计时器」设长或无限。
- **监看帧率不高 / HD 更低**：WiFi 拉取式取帧，VGA 约 10–20fps、XGA 更低，属正常。
- **预览用 RGB_565**：内存减半，屏幕上与全彩无肉眼差别。
- **放大时不能翻页**：主流相册一致行为，缩回 1x 才能翻页。
