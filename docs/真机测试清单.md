# Z传 — 真机回归测试清单

> 覆盖：连接 / 浏览 / 缩略图 / 传输 / 掉线续传 / 取消重试 / 设置。
> 每项含**操作**、**预期**、**判定**。文末「设计如此，非 Bug」一节列出几个看起来可能像问题、但属预期的表现——测试时请对照，避免误报。

图例：⬜ 待测　✅ 通过　❌ 不符（记录现象）

---

## 1. 连接相机

### 1.1 先连 Wi-Fi，再开 App
- 操作：手机连上相机 Wi-Fi → 打开 App。
- 预期：首页短暂显示"正在连接相机…"→ 连上后自动跳转文件列表并开始加载。
- ⬜

### 1.2 先开 App，再连 Wi-Fi
- 操作：先打开 App（未连相机 Wi-Fi）→ 再去系统设置连相机 Wi-Fi → 回到 App。
- 预期：≤3 秒内自动开始连接并进入文件列表（网络回调或 3s 看护循环触发）。
- ⬜

### 1.3 相机未开机/未就绪时
- 操作：在相机 Wi-Fi 上，但相机 PTP 尚未就绪。
- 预期：**一直显示"正在连接相机…"、不弹错误**，相机就绪后自动连上。
- ⬜

### 1.4 断开相机 Wi-Fi（切到别的网/关 Wi-Fi）
- 操作：连上后，手机切换到其它 Wi-Fi 或关闭 Wi-Fi。
- 预期：状态变为未连接；切回相机 Wi-Fi 后自动重连。
- ⬜

## 2. 文件列表（列表模式，缩略图开关关闭）

### 2.1 分组与流式加载
- 预期：按日期分组，组内新→旧；文件边加载边出现（底部"正在加载更多…"）；每项显示类型角标(JPG/NEF/MOV)、文件名、大小。
- ⬜

### 2.2 空相机
- 预期：显示"相机中无文件"。
- ⬜

---

## 3. 缩略图模式（本次新增）

### 3.1 开关
- 操作：设置 → 打开"缩略图模式"。
- 预期：文件列表切换为网格缩略图；关闭后回到文本列表。分组方式不变。
- ⬜

### 3.2 列数 1–4
- 操作：设置里切换每行 1/2/3/4 列。
- 预期：网格实时按所选列数排布；重启 App 后设置保留。
- ⬜

### 3.3 缩略图加载
- 预期：进入网格后，可见缩略图逐个加载出图（滚动时新可见项继续加载）；已加载的滚回来**秒显**（走缓存）。取不到缩略图的显示占位图标 + 类型角标。
- ⬜

### 3.4 缩略图上的传输状态
- 操作：点一张图入队。
- 预期：该格出现半透明遮罩 + 右下角状态角标（等待/传输中/完成/失败/取消）。
- ⬜

### 3.5 缩略图让路传输（关键）
- 操作：网格模式下，把一个分组加入传输队列，传输进行中滚动浏览未加载过的缩略图。
- 预期：**传输期间不加载新缩略图**（已加载的仍显示），传输速度不受缩略图影响；**队列全部完成后，未加载的缩略图自动补上**。
- ⬜

---

## 4. 传输

### 4.1 单文件 / 整组
- 操作：点单张图；或点分组"传输"按钮。
- 预期：跳到/可进入传输页；任务按序下载，显示已下载/总大小、速度、完成计数；完成变绿。
- ⬜

### 4.2 进度与速度
- 预期：传输中进度条与速度实时更新（约每 200ms）；顶栏药丸显示 `已完成/总数` 与速度。
- ⬜

### 4.3 文件落地与命名
- 操作：传输后到所选目录查看。
- 预期：文件完整可打开（JPG 能看图、NEF/MOV 大小正常）；无重名副本、无残缺文件。
- ⬜ **重点：对比相机原图，确认无截断（可对 1 张大文件比对大小/能否正常打开）。**

### 4.4 已存在则跳过
- 操作：传完一批后，再次把同一批加入传输（或对已传过的文件再点传输）。
- 预期：目标目录已有的同名文件**直接跳过**（传输页显示"已存在"、状态为完成），**不重复下载、不产生 "(1)" 副本**；只有目录里没有的才真正下载。
- ⬜

### 4.4 后台/锁屏保活
- 操作：传输中按 Home 键回桌面或锁屏，等待。
- 预期：通知栏出现"正在传输文件…"；传输继续不中断；完成后通知消失。
- ⬜

## 5. 未设置传输目录

### 5.1 未设目录点传输
- 操作：从未设过目录，点单图或分组"传输"。
- 预期：**直接跳转到设置页**（不入队、不产生失败任务）。
- ⬜

### 5.2 设置目录后
- 操作：设置页选目录 → 返回列表再点传输。
- 预期：正常入队并开始传输。
- ⬜

---

## 6. Wi-Fi 掉线 / 重连续传（关键）

### 6.1 传输中断开相机 Wi-Fi
- 操作：正在传输时，手机切走/关闭 Wi-Fi。
- 预期：进行中的任务**很快**停下（不会等很久）；**剩余未传任务保持"等待"，不会一个个变失败**；不弹连接错误。
- ⬜

### 6.2 重连后自动续传
- 操作：接 6.1，重新连回相机 Wi-Fi。
- 预期：自动重连 → **剩余"等待"任务自动继续传输**，直到全部完成。中断当时那一张会重新完整传（非断点续传）。
- ⬜

### 6.3 相机关机再开机（Wi-Fi 层面掉线）
- 操作：传输中关掉相机，稍后重开并重连 Wi-Fi。
- 预期：同 6.1/6.2——暂停 → 重连 → 续传。
- ⬜

### 6.4 空闲掉线
- 操作：连着但不传输，断开相机（关机/走远）。
- 预期：≤10 秒内状态变为未连接；恢复后自动重连并重载列表。
- ⬜

---

## 7. 取消 / 重试

### 7.1 取消传输
- 操作：传输页/相应入口取消。
- 预期：等待中与进行中的任务变"已取消"；正在写的半成品文件被删除（目录里不残留）。
- ⬜

### 7.3 崩溃/被杀后的半成品清理
- 操作：传输一个大文件（视频）中途，用系统「最近任务」划掉 App / 强制停止，制造"进程被杀"。
- 预期：目标目录里**不会出现残缺的真名文件**；只可能残留一个隐藏的 `.nkpart_xxx` 临时文件；**下次打开 App 或再次传输时会自动清扫掉**。真名文件一旦出现即为完整。
- ⬜

### 7.2 重试单个/失败项
- 操作：对失败/取消的任务点"重新传输"。
- 预期：该任务重新入队并传输；成功后变完成，目录里无重名副本。
- ⬜

## 设计如此，非 Bug（对照，避免误报）

1. **初始加载时缩略图较晚出现**：文件信息与缩略图共用同一条 PTP 通道，串行进行。列表信息加载期间缩略图会等列表加载告一段落再陆续出图，属正常。
2. **重连后文件列表会"重载一下"**：为确保新会话数据一致（句柄/存储可能变化），重连后会清空并重新加载文件列表；此时若有暂停的传输，会在列表重载后继续——短暂等待属预期。
3. **传输进行中滚动看到占位、暂不出图**：这是"缩略图让路传输"的设计，队列清空后自动补载。
4. **连接失败不弹错误、一直转圈**：按需求改为"只要在相机 Wi-Fi 上就持续重试直到连上"，不再三次后报错。
5. **续传是整文件重传**：PTP 无通用断点续传，中断的那张会从头重传，最终文件正确。
6. **不在相机 Wi-Fi 上不主动连**：off-wifi 时无到相机 IP 的路由，App 只做廉价的本地检测、不空转 socket（省电）；进入相机 Wi-Fi 即自动连。
