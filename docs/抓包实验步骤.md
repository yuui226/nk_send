# 「连接到手机热点」配对握手 —— 抓包实验操作手册

> 配套脚本:[`tools/nikon_ptpip_probe.py`](../tools/nikon_ptpip_probe.py)
> 目标:一次实验拿全两样东西 —— ①WTU 官方配对的完整报文(用来逆向认证码握手);②相机的 DeviceInfo(操作码/属性码/机型/序列号)。
> 需要的东西:一台 Windows(或 Mac)电脑、Z50 或 Z30 真机、Wireshark、装了 `zeroconf` 的 Python。

---

## 零、准备(5 分钟)

```bat
python -m pip install zeroconf
```

- 装 Wireshark(https://www.wireshark.org/,Windows 版自带 Npcap,安装时勾选)。
- **关掉/放行防火墙**:Windows 防火墙默认可能挡 TCP 15740 与 UDP 5353。实验期间可临时关私有网络防火墙,或给 WTU 和 python.exe 放行。
- 记下相机 MAC:相机菜单 **网络/连接 → 连接到电脑 → 网络设置 → (对应配置文件) → 选项 里有 MAC address**;先抄下来,抓包时用它当过滤锚点最省事。

## 一、组网:让相机进「连接到电脑」模式并入网

本次实验用**手机热点**(和最终产品一致),电脑也连到同一个热点:

1. 手机打开个人热点,安全性设为 **WPA2**(或 WPA2/WPA3 兼容),记下 SSID/密码。
2. **电脑连上这个手机热点**(WTU 和相机必须在同一网段)。
3. 相机上:**连接到电脑 → 网络设置 → 创建配置文件 → 搜索 Wi-Fi 网络 → 选手机热点 → 输密码**,选 **基础结构模式(infrastructure/station)**,不要选相机自身做接入点。
4. 连上后相机会显示分到的 IP(记下来,后面 `--ip` 直接用最稳)。

> 若现场只能用公司路由器也行,原理一样;但公司网设备多,务必靠脚本的三重信号(mDNS + 15740 + 握手回名)定位相机,别靠猜 IP。

## 二、先跑探针,确认能定位到相机(不碰 WTU)

在电脑上先单独跑脚本,验证「精准识别」这一步,并顺手把 DeviceInfo 拿到手:

```bat
cd tools
:: 已知 IP 时(推荐):
python nikon_ptpip_probe.py --ip 192.168.x.x
:: 或让它自己在热点网段发现:
python nikon_ptpip_probe.py --subnet 192.168.x.0/24
```

预期两种结果,都对我们有用:

- **Init 成功 + 回读到「Z50」** → 说明这台机器在「连接到电脑」模式下**对任意 GUID 放行**(和老机型「智能设备」模式一样),那我们几乎不用逆向配对,产品实现成本极低。
- **Init Fail / OpenSession 被拒(0x2019、0x201E 之类)或相机屏幕弹出认证码等待** → 说明**必须先配对**,这才需要下一步抓 WTU。脚本会把 `fail_reason` 打出来并写进 JSON。

无论哪种,脚本都会生成 `ptpip_probe_<时间戳>.json`,把它留好。

## 三、抓包 WTU 官方配对(核心一步)

只有当第二步表明「需要配对」时才必须做这步;但**建议无论如何都抓一次**,作为握手基准。

1. 打开 Wireshark,选中**电脑连手机热点的那个无线网卡**接口。
2. 抓包前先在顶部 capture filter 里填(缩小范围、减噪):
   ```
   host <相机IP> or udp port 5353
   ```
   若还不知道相机 IP,用 MAC:`ether host <相机MAC> or udp port 5353`。
3. 点开始抓包(鲨鱼鳍)。
4. **在相机上删除刚才那台电脑的已配对记录**(网络设置里),确保这次是「首次配对」,能抓到完整认证码交换。
5. 打开 **Wireless Transmitter Utility → 添加设备/配对**,让它发现相机 → 相机屏幕显示认证码 → 在 WTU 里输入 → 完成配对。
6. 配对成功后再让 WTU/相机跑一次实际传输(拍一张、自动上传一张),多抓一点正常流量。
7. 停止抓包,**另存为 `.pcapng`**(File → Save As)。

抓包时的显示过滤器(分析用,不影响已抓数据):
```
tcp.port == 15740 || udp.port == 5353
```
- `udp.port == 5353` 看 mDNS:能确认尼康用的**服务名**(是不是 `_ptp._tcp`)和相机主机名。
- `tcp.port == 15740` 看 PTP/IP:Init Command Request/Ack、认证码在哪个包、配对用什么字段确认。追一条 TCP 流:右键任一 15740 包 → **Follow → TCP Stream**,能把整段握手看全。

## 四、把结果发我

打包这三样发我即可分析并给出 Kotlin 实现方案:

1. `tools/ptpip_probe_<时间戳>.json`(第二步脚本产物);
2. `配对.pcapng`(第三步抓包);
3. 相机型号 + 固件版本 + 你观察到的现象(认证码是几位、WTU 里是否要手动选相机等)。

## 常见问题

| 现象 | 处理 |
|---|---|
| 脚本扫不到任何设备 | 电脑和相机不在同一网段 / 防火墙挡了 15740;先 `ping 相机IP` 通不通 |
| 扫到一堆 15740 命中 | 公司网里可能有别的 PTP 设备;看脚本「握手回读 FriendlyName」那台才是相机 |
| mDNS 无结果 | 没装 zeroconf,或热点接口 mDNS 被隔离;不影响,靠 `--ip` / 端口扫描即可 |
| WTU 发现不了相机 | 关电脑防火墙;确认相机停在「连接到电脑 · 等待连接」界面;两者同网段 |
| Wireshark 看不到 15740 流量 | 选错网卡接口了;抓包接口要选**连手机热点的那张网卡** |
| 相机连热点后掉线 | 热点设 WPA2、2.4/5GHz 都试;部分相机不认 WPA3 |
