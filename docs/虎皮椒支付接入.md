# 虎皮椒自动售码 —— 接入说明(当前方案:微信扫码)

> 目标:App 内点"立即购买"→ 显示微信收款二维码 → 用户微信扫码付款 → 服务器确认到账自动发码 → App 自动激活。
> 渠道:虎皮椒 https://www.xunhupay.com **微信个人支付**(个人实名 + 真实业务网站即可签约,无执照/门头照门槛)。
> 备注:支付宝渠道已被虎皮椒/支付宝收口走不通(2026-07),故改走微信;签约域名用 https://www.ztransfer.top(GitHub Pages)。
> 微信个人支付**不支持 App 内直接唤起微信**,故 App 端展示二维码引导用户扫码(见流程)。
> 成本:开通费 ¥118(支付宝已付 ¥88,补差价 ¥30)+ 官方 0.6% + 平台费 1–2%,合计约 2.6%/笔。
> 资金安全:微信财付通 D+1 清算到本人银行卡,**平台不碰货款**。
> 代码已完成(分支 feature/auto-pay),填入密钥即可上线。API 文档:https://www.xunhupay.com/doc

## 一、待办(人工步骤)

- [ ] **1. 注册虎皮椒 + 开通微信个人支付**:https://www.xunhupay.com 实名签约,签约域名填 https://www.ztransfer.top。
- [ ] **2. 拿到凭据**:后台微信渠道的 `appid` 与 `appsecret`。
- [ ] **3. 服务器配置**:config.json 增加(四项齐全才启用,缺任一则购买接口返回 PAY_DISABLED):
  ```json
  {
    "xhAppId":      "<后台 appid>",
    "xhAppSecret":  "<后台 appsecret>",
    "priceFen":     2900,
    "payNotifyUrl": "https://106.15.239.203:8443/pay/notify"
  }
  ```
  `priceFen` 为售价(分,接口侧自动转元)。
- [ ] **4. 部署**:上传新 license-server.js → `systemctl restart ztransfer-license`,
  启动日志确认 `pay=xh ¥xx`。
- [ ] **5. App 定价展示**:SettingsScreen.kt 的 `PRO_PRICE` 填定价文案(如 "¥29"),与 `priceFen` 一致,发版。
- [ ] **6. 真机联调**(拿到微信 appid 后重点验证):小额真付一单走通 购买→显示二维码→微信扫码付款→到账→自动激活→展示码。
  **需现场确认**:①`url_qrcode` 是否为可直接加载的图片 URL(应是,文档"不用再转二维码");
  ②同机扫码路径是否顺(截图→微信扫一扫→相册)。若行为不符再微调 PurchaseDialog。

## 二、流程设计

```
App「立即购买」(ProDialog 金色按钮)
  → POST /v1/order/create {fp}          服务器调虎皮椒 /payment/do.html 建单
  → 返回 {order, pay_qr, pay_url}       pay_qr = 现成的微信收款二维码图片地址;pay_url = 手机端链接(降级)
  → PurchaseDialog 加载 pay_qr 图片显示  引导:截图 → 微信「扫一扫」→ 相册选图 付款(或另一台手机直接扫)
  → 用户微信付款
  → App 每 2s POST /v1/order/status      服务器懒查虎皮椒 query(4.5s 节流)
  → status=OD(已支付)→ 发码             幂等:一单永远只发同一个码,note 记 xh:订单号
  → App 拿码自动走 /v1/activate 激活      并展示激活码让用户截图留底(一码两机)
```

可靠性:

- **断点续单**:App 本地记 `pending_order`;付款后 App 被杀,重开购买页查旧单补发码。
  待支付旧单超 15 分钟不再复用收银台链接(链接短时效),自动落回新建订单。
- **notify 验签**:虎皮椒回调带 MD5 签名,验签通过才处理;发码仍统一走主动查单的单一路径,
  回调只是加速信号,处理完回纯文本 `success` 停止重推(最多重推 6 次)。
- **退款**:回调 status=CD/RD/UD 记日志人工跟进,配合 `/admin/revoke` 吊码;
  orders 表(code ↔ charge_id=虎皮椒单号)即对账台账。
- **防刷**:下单与激活共用每 IP 每分钟 10 次限速;同设备 15 分钟内未支付订单复用;7 天死单自动清理。
