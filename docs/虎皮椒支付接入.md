# 虎皮椒自动售码 —— 接入说明(当前方案)

> 目标:App 内点"立即购买"→ WebView 打开虎皮椒收银台唤起支付宝付款 → 服务器确认到账自动发码 → App 自动激活,零人工。
> 渠道:虎皮椒 https://www.xunhupay.com (个人实名即可签约,无执照/照片/粉丝门槛)。
> 成本:开通费 ¥118/渠道(一次性)+ 官方 0.6% + 平台费 1–2%(预存),合计约 2.6%/笔。
> 资金安全:微信财付通 D+1 清算到卡 / 支付宝直达自有账户,**平台不碰货款**——平台倒闭最多损失预存平台费。
> 代码已完成(分支 feature/auto-pay),填入密钥即可上线。API 文档:https://www.xunhupay.com/doc

## 一、待办(人工步骤)

- [ ] **1. 注册虎皮椒**:https://www.xunhupay.com 实名签约,先开通**支付宝**渠道(¥118);
  微信渠道视需求后补(App 端收银台/唤起逻辑已兼容两者)。
- [ ] **2. 拿到凭据**:后台的 `appid` 与 `appsecret`。
- [ ] **3. 服务器配置**:config.json 增加(四项齐全才启用,缺任一则购买接口返回 PAY_DISABLED):
  ```json
  {
    "xhAppId":      "<后台 appid>",
    "xhAppSecret":  "<后台 appsecret>",
    "priceFen":     2900,
    "payNotifyUrl": "https://106.15.239.203:8443/pay/notify"
  }
  ```
  `priceFen` 为售价(分,接口侧自动转元)。
- [ ] **4. 部署**:上传新 license-server.js → `systemctl restart ztransfer-license`,
  启动日志确认 `pay=xh ¥xx`。
- [ ] **5. App 定价展示**:SettingsScreen.kt 的 `PRO_PRICE` 填定价文案(如 "¥29"),与 `priceFen` 一致,发版。
- [ ] **6. 真机联调**:小额真付一单走通 购买→到账→自动激活→展示码,然后后台退款(或留作首单)。

## 二、流程设计

```
App「立即购买」(ProDialog 金色按钮)
  → POST /v1/order/create {fp}          服务器调虎皮椒 /payment/do.html 建单
  → 返回 {order, pay_url}               pay_url 为收银台 H5 链接
  → PurchaseDialog 内 WebView loadUrl    收银台自动唤起支付宝(alipays:// 由 App 转交系统)
  → 用户付款,切回 App
  → App 每 2s POST /v1/order/status      服务器懒查虎皮椒 query(2.5s 节流)
  → status=OD(已支付)→ 发码             幂等:一单永远只发同一个码,note 记 xh:订单号
  → App 拿码自动走 /v1/activate 激活      并展示激活码让用户截图留底(一码两机)
```

可靠性:

- **断点续单**:App 本地记 `pending_order`;付款后 App 被杀,重开购买页查旧单补发码。
  待支付旧单超 15 分钟不再复用收银台链接(链接短时效),自动落回新建订单。
- **notify 验签**:虎皮椒回调带 MD5 签名,验签通过才处理;发码仍统一走主动查单的单一路径,
  回调只是加速信号,处理完回纯文本 `success` 停止重推(最多重推 6 次)。
- **退款**:回调 status=CD/RD/UD 记日志人工跟进,配合 `/admin/revoke` 吊码;
  orders 表(code ↔ charge_id=虎皮椒单号)即对账台账。
- **防刷**:下单与激活共用每 IP 每分钟 10 次限速;同设备 15 分钟内未支付订单复用;7 天死单自动清理。
