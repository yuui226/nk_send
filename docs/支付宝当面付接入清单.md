# 支付宝当面付接入 —— 待办清单

> 目标:用户在 App 内点"购买"→ 唤起支付宝付款 → 服务器自动发激活码,发码环节零人工。
> 费率 0.38%,货款实时进个人支付宝余额。个人无执照可申请(基础版限额:单笔 ≤1000 元、日 ≤5 万)。
> 技术方案概要见文末;详细激活体系见 [激活与付费设计.md](激活与付费设计.md)。

## 一、申请开通(人工步骤,先做,审核 1–3 个工作日)

- [ ] **1. 入驻开放平台**:用实名支付宝账号登录 https://open.alipay.com ,入驻身份选"自研开发者"(个人)。
- [ ] **2. 创建应用**:控制台 → 创建"网页/移动应用",名称可写 "ZTransfer 激活服务"。记下 **APPID**。
- [ ] **3. 拍经营场所照片**:没有店面就拍**工作场景**——书桌 + 电脑,屏幕上开着 ZTransfer 或代码,照片清晰、无水印、无 PS。这是审核唯一需要准备的材料。
- [ ] **4. 签约当面付**:应用详情 → 产品绑定 → 添加"当面付"→ 提交签约。
  - 经营类目:选普通零售类(避开金融、虚拟充值等敏感类目)
  - 经营描述:如"自研软件工具销售"
  - 上传第 3 步的照片
- [ ] **5. 等审核**(1–3 个工作日)。被拒最常见原因是照片不符,按驳回意见补拍重提即可,不花钱。
- [ ] **6. 配置密钥**(审核通过后):
  - 用支付宝官方"密钥工具"(或 openssl)生成 RSA2 2048 密钥对
  - **应用公钥**填到开放平台 → 平台回显一把**支付宝公钥**(验签用),保存下来
  - **应用私钥**放到服务器(与 signingKey 同级,严禁进 git)
  - config.json 将新增:`alipayAppId` / `alipayPrivateKeyPath` / `alipayPublicKeyPath` / `priceYuan`
- [ ] **7. (可选)开通沙箱**:开放平台 → 沙箱环境,有独立网关和沙箱支付宝 App,审核期间即可并行联调。

## 二、若个人申请被拒的备选

- **办个体工商户执照**(一劳永逸):本省政务服务网 → "个体工商户设立登记",住宅地址可用(社区盖章"住改商"),经营范围写"软件开发;软件销售",1–3 天拿电子执照,免费。有执照后当面付秒批 + 解除限额,还可开通微信支付官方通道。年报一年一次;小规模月销 10 万内免增值税。
- **面包多 Pay 兜底**(不碰审核):https://p.mbd.pub 个人身份证即可开通,卡密自动发货,每单抽 2% + 0.1 元。

## 三、开发改动(申请提交后即可动工,约 2 天)

### 服务端 license-server.js(~1 天,继续零依赖)

- 新表 `orders`:`out_trade_no PK, device_fp, amount, status(pending/paid/expired), code, alipay_trade_no, created_at, paid_at`
- 签名/验签用 `node:crypto` 直接实现(参数排序拼串 → SHA256withRSA),不引 SDK
- 三个新端点:
  | 端点 | 作用 |
  |---|---|
  | `POST /v1/order/create` | 生成 out_trade_no → 调 `alipay.trade.precreate` → 返回 qr_code(复用现有 IP 限速) |
  | `GET /v1/order/status` | pending 时懒查 `alipay.trade.query`;首次确认已付 → newCode() 发码(note 记 `alipay:交易号`),幂等 |
  | `POST /alipay/notify` | 验签 + 校验金额/app_id → 标记已付 → 回纯文本 `success` |
- 支付确认以**主动查询为主链路**(服务器在 8443 非标端口,异步通知不保证可达),notify 只是加速
- 订单设 `timeout_express=15m`,过期作废
- 退款流程:`alipay.trade.refund` + 现有吊销接口 revoke 对应激活码,orders 表即台账

### App 端(~0.5–1 天)

- 付费引导页:"加 QQ 群"引导 → "支付宝购买 ¥xx"按钮
- 点击 → `/v1/order/create` → 唤起 `alipays://platformapi/startapp?saId=10000007&qrcode=<urlencode(qr_code)>`;未装支付宝则本地渲染二维码图
- `onResume` 起每 2 秒轮询 `/v1/order/status`,拿到码后自动走现有激活流程,并弹窗展示激活码让用户留底(一码两机,第二台设备手输)
- 兜底:用户付了但 App 被杀 → 下次打开付费页按 device_fp 补领最近已付订单的码

### 安全底线

- 一切以支付宝侧 trade.query / 验签通过的 notify 为准,不信任 App 上报
- notify 必须校验 `total_amount` 与订单一致、`app_id` 是自己的
- 同一 out_trade_no 永远只发同一个激活码(幂等)
