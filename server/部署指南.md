# 激活服务器部署指南(阿里云 ECS,IP 直连,免域名)

整套服务零 npm 依赖,只需要 Node ≥ 22.13(`node:sqlite` 要求)。

## 1. 装 Node(二进制包,不依赖发行版源)

```bash
curl -fsSL https://npmmirror.com/mirrors/node/v22.20.0/node-v22.20.0-linux-x64.tar.xz -o /tmp/node.tar.xz
mkdir -p /opt/node && tar -xJf /tmp/node.tar.xz -C /opt/node --strip-components=1
/opt/node/bin/node --version   # 应显示 v22.20.0
```

## 2. 部署文件与初始化

```bash
useradd -r -s /sbin/nologin ztlicense
mkdir -p /opt/ztransfer-license && cd /opt/ztransfer-license
# 把 license-server.js / setup-keys.js 传到这个目录(scp / 宝塔 / xftp 均可)

/opt/node/bin/node setup-keys.js init .
# 输出两样东西,都要保存:
#   - App 内置公钥(写进 App 的 License.kt)
#   - config.json(含 adminToken,发码脚本要用)

# 生成自签名 TLS 证书(把 <ECS公网IP> 换成真实 IP,有效期 10 年)
openssl req -x509 -newkey ec -pkeyopt ec_paramgen_curve:prime256v1 \
  -keyout tls-key.pem -out tls-cert.pem -days 3650 -nodes \
  -subj "/CN=ztransfer-license" -addext "subjectAltName=IP:<ECS公网IP>"

/opt/node/bin/node setup-keys.js pin tls-cert.pem
# 输出证书指纹(写进 App 的 License.kt)

chown -R ztlicense:ztlicense /opt/ztransfer-license
chmod 600 signing-key.pem tls-key.pem config.json
```

**备份 `signing-key.pem` 和 `config.json` 到你自己的电脑**(丢了私钥 = 已售激活码全部无法续签)。

## 3. systemd 常驻

```bash
cp ztransfer-license.service /etc/systemd/system/
systemctl daemon-reload
systemctl enable --now ztransfer-license
systemctl status ztransfer-license      # 应显示 listening on :8443
journalctl -u ztransfer-license -f      # 看实时日志
```

## 4. 阿里云安全组

- 入方向放行 **TCP 8443**(来源 0.0.0.0/0,App 要用)。
- 可选加固:`/admin/*` 已有 token 鉴权,若想双保险,可再建一条只放行你家宽带 IP 的规则并让
  管理走别的端口——初期不必。

## 5. 验证

```bash
curl -k https://<ECS公网IP>:8443/healthz     # -> {"ok":true}
```

在你的 Windows 电脑上:**双击 `激活码管理.bat`** 进入交互菜单(首次会提示输入
config.json 里的 adminToken,自动保存到同目录 `admin-token.txt`,该文件等同密码勿外传)。
菜单里可以生成激活码、看台账、解绑、吊销、测试连接。

也可以命令行调用(令牌来源同上,或设 `$env:ZT_ADMIN_TOKEN`):

```powershell
.\admin.ps1 new 1 "测试批次"     # 生成第一个激活码
.\admin.ps1 list                # 查看台账
```

## 6. 日常运维

日常操作直接双击 `激活码管理.bat` 用菜单即可;等价的命令行:

| 事项 | 操作 |
|------|------|
| 发码 | `.\admin.ps1 new <数量> "<备注>"` |
| 用户换机 | `.\admin.ps1 unbind <激活码> <旧设备码 ZT-XXXX-XXXX>` |
| 退款吊销 | `.\admin.ps1 revoke <激活码> "<原因>"`(该码全部设备在下次续签时降级) |
| 备份 | 每天 cron:`cp /opt/ztransfer-license/license.db /opt/backup/license-$(date +%F).db` |
| 换服务器/IP | 迁走 signing-key.pem + license.db + config.json;App 发新版更新内置 IP 列表 |

## 7. App 检查更新(app-latest.json)

App 连接页的「检查更新」走 `GET /v1/app/latest`,服务器现读 config.json 同目录的
`app-latest.json`(仓库里有模板)。**发新版流程**:APK 传蓝奏云 → 改这个文件,无需重启服务:

```json
{
  "versionCode": 3,            // 与新版 build.gradle.kts 的 versionCode 一致(App 按它比大小)
  "versionName": "1.4",        // 展示用版本名
  "url": "https://……",         // 蓝奏云分享链接(建议建"无提取码"的分享)
  "password": "",              // 提取码;留空 = 无码。有码时 App 点"去下载"自动复制
  "notes": "修复了……"           // 更新说明,显示在对话框里,可留空
}
```

文件缺失/字段不全时接口返回 NO_VERSION_INFO,App 侧按"检查失败"提示,不影响其它功能。
